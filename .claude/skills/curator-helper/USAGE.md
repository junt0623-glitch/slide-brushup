# USAGE.md — curator-helperの使い方

## 自動発火する依頼例
以下のような依頼をすると、Claude Codeが自動的にこのスキルを読み込みます。特別なコマンドを打つ必要はありません。

- 「この青磁の花瓶のキャプション書いて。南宋時代、高さ18cm」
- 「来月の展覧会企画書のたたき台作って」
- 「助成金申請の事業概要、もっと審査員に刺さる書き方にできない?」
- 「プレスリリース、まだ何も書いてなくて今日中に出したい」
- 「こどもワークショップの資料作って、対象は小学生低学年」

## 手動で呼び出す方法
名前を直接指定したい場合は、依頼の中に「curator-helperを使って」と一言加えるか、Claude Codeで `/curator-helper` のように呼び出してください(スラッシュコマンドとしての挙動は環境により異なります)。

## 発火しない依頼の例
以下は対象外です(brief.mdの「今回やらないこと」参照):
- 「この写真の色調を明るく補正して」→ 画像編集
- 「予算のシミュレーション組んで」→ 会計・予算管理
- 「この壺、本物か鑑定して」→ 画像からの真贋鑑定

## 初回テストのやり方
1. Claude Codeを再起動する
2. `evals/trigger-evals.json` の「should_trigger」の例文を1つ試し、スキルが読み込まれるか確認する
3. `evals/output-evals.json` の「通常ケース」のプロンプトを試し、出力が `acceptance-criteria.md` を満たしているか確認する

## 文字数チェックスクリプトの使い方
```bash
python3 scripts/check_length.py --text "本文をここに" --limit 200
```
`passed: true` なら制限内、`false` なら超過しています。
