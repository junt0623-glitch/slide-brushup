# acceptance-criteria.md — curator-helper

各成果物は、公開・提出前に以下をすべて満たすこと。

## Hard Gates(1つでも不合格なら差し戻し)
- [ ] でっち上げの出典・文献・年代・展覧会歴がない
- [ ] 未確認情報は「要確認」「推定」と明記されている
- [ ] 指定された文字数・字数制限を超えていない(キャプション等)
- [ ] 依頼された言語(日本語/英語)で出力されている

## Defaults(理由があれば変更可)
- [ ] 一般来館者向け文章は、専門用語に一行説明が添えてある
- [ ] キャプションは「作品情報→見どころ→背景」の順で構成されている
- [ ] 助成金・プレスリリースは、当該フォーマットの慣例的な構成(宛先/概要/詳細/連絡先など)に沿っている

## Preferences(余裕があれば最適化)
- [ ] 文末が単調にならない(「〜です。」の連続を避ける)
- [ ] 紋切り型の褒め言葉(「魅力あふれる」「必見」等)に頼らず、具体的な特徴描写になっている
- [ ] 読み手が次に取るべき行動(鑑賞のポイント、申請の次のステップ等)が明確

## 検証方法
- 文字数制限があるものは `scripts/check_length.py` で機械的に確認する
- でっち上げ疑惑がある固有名詞・年代は、本文中の入力情報だけから説明できるか目視で追跡する
