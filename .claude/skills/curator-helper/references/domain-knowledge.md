# domain-knowledge.md

いつ読むか: 依頼された文書の種類が決まった直後に、該当セクションだけ読む。
何を判断するために読むか: その文書タイプ特有の構成・慣例・注意点を反映するため。

---

## 1. 作品キャプション(館内解説・図録テキスト)

構成の基本形:
1. 作品情報(作家・時代・技法・材質・寸法・所蔵)— 事実の羅列は簡潔に
2. 見どころ(視覚的特徴、技法上の特筆点)— ここが本文の中心
3. 背景(制作背景、来歴、文化的文脈)— 紙幅が許す範囲で

注意点:
- 一般来館者向けは専門用語(例: 「窯変」「貫入」)に一行説明を添える
- 図録用(専門家向け)は用語説明を省略してよいが、その旨をユーザーに確認する
- 文字数制限は必ず守る。指定がなければ assumptions.md の既定値(約200字)を仮の目安にし、その旨を出力に明記する

## 2. 来歴(provenance)調査メモ

- 事実(誰が所有し、いつ、どのように移転したか)と推測を明確に分けて記述する
- 未確認の伝承は「〜と伝えられる」「〜という記録があるが未確認」と書く
- 出典が分かっている場合は本文中の入力情報の範囲でのみ引用元を示す。存在しない文献名を作らない

## 3. 展覧会企画書・コンセプト文

構成の基本形:
1. テーマと開催意義(なぜ今この展覧会か)
2. 構成(章立て・見どころ作品)
3. 対象来館者と期待される効果
4. 会期・会場等の実務情報(分かっている範囲で)

注意点:
- 「〜という魅力があります」で終わる感想文にしない。具体的な作品・データ・視点で裏付ける

## 4. 助成金申請書

構成の基本形(一般的な慣例。実際の様式が指定されていればそちらを優先):
1. 事業概要
2. 目的・意義
3. 実施内容・スケジュール
4. 期待される成果・効果測定方法
5. 予算(具体額はユーザーから提供された場合のみ記載、なければ「要記入」とする)

注意点:
- 審査員は多くの申請書を読むため、冒頭2〜3行で概要が伝わる書き出しにする
- 数値目標(来館者数等)はユーザーが提供した情報のみを使い、勝手に数字を作らない

## 5. プレスリリース

構成の基本形:
1. 見出し(要点が一目で分かる)
2. リード文(5W1Hを簡潔に)
3. 詳細本文
4. 開催概要(日時・場所・料金等)
5. 問い合わせ先(プレースホルダーでよい)

## 6. 教育普及プログラム資料

- 対象年齢・対象層を明確にする
- ワークショップ等は「ねらい」「流れ(時間配分)」「必要物品」を分けて書く
- 安全面・アクセシビリティへの配慮に触れる
