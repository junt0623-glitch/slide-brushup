モデル役割分担ポリシー
0. 基本原則
目的：品質を落とさずトークン消費を最小化する。
原則：「考える工程」は上位モデル、「手を動かす工程」は中位、「定型作業」は軽量モデルに委譲する。
上位モデルの成果物は必ず**下位モデルへの指示書（設計書・仕様書・チェックリスト）**の形にし、下位モデルが迷わず実行できる粒度まで具体化してから引き渡す。

2. 困難度の判定軸（4軸）
タスク着手前に以下4軸で困難度を判定し、階層を選ぶ。

影響範囲：変更が全体構造に波及するか、局所で閉じるか
曖昧さ：要件が固まっているか、解釈・調査・判断が必要か
失敗コスト：誤りが破壊的か（データ破損・後戻り困難）、やり直しが容易か
推論の深さ：多段の因果推論・トレードオフ比較が必要か、手順の実行で足りるか

判定ルール：2軸以上が「高」→ Tier 1／1軸のみ「高」→ Tier 2／すべて「低」→ Tier 3。迷ったら一つ上の階層を選ぶ（誤配置による手戻りの方がトークンを浪費するため）。

4. トークン節約ルール
コンテキスト最小化：各階層には該当タスクに必要なファイル・差分のみを渡す。リポジトリ全体を読ませない。
一往復主義：Tier 1 への相談は論点をまとめて一度で行う。細切れの質問を繰り返さない。
成果物の再利用：Tier 1 が作った設計書・チェックリストは保存し、類似タスクで再利用する。
サブエージェント活用：探索・調査系のサブエージェントは Tier 2/3 に固定する（環境変数 CLAUDE_CODE_SUBAGENT_MODEL、またはサブエージェント定義の frontmatter model: で指定）。
完了条件：出力は実際のテスト実行で検証されて初めて完了とする。検証はテスト実行そのものなので Tier 3 でよいが、失敗時の原因分析は判定軸に従って昇格させる。
5. 切り替え方法（操作手順）
セッション中の切り替え：チャット欄に /model claude-fable-5 のように入力（引数なしの /model でピッカー表示）
セッション開始時：claude --model claude-sonnet-4-6
プロジェクト既定値：.claude/settings.json の model キーで指定
注意：Fable 5 は Claude Code v2.1.170 以降が必要。表示されない場合は claude update を実行
6. フォールバック条項（重要）
claude-fable-5 が従量課金制に移行した場合、以下を即時適用する：

筆頭（Tier 1）を claude-opus-4-8 に置き換える
§2〜§5 の采配・判定軸・節約ルールは一切変更せず同一のまま運用する
Fable 5 は使用しない（明示的な指示がある場合のみ例外）
置き換え作業：本文書内の claude-fable-5 を claude-opus-4-8 に読み替えるだけでよい
