# スライドブラッシュアップ工房 ─ Phase 1

pptxインポート・解析・ハッシュ台帳・サムネイル表示。設計書§7のPhase 1に対応。

## できること

- `.pptx` をドラッグ&ドロップ（またはクリック選択）で解析
- 全スライドのテキスト・画像・位置・表・発表者ノートを抽出
- スライド順は**ファイル名ではなくrelationships経由で正しく解決**（PowerPointが
  スライド並べ替え後にファイル名を飛び番で保存しても正しく動く）
- 画像はバイト単位で無改変のまま保持し、同一画像は自動的に重複排除
- テキスト・画像それぞれにSHA-256ハッシュを保存（改変検出の基盤。実際の照合は
  Phase 2の書き出し検証で使う）
- グラフ／SmartArt／埋め込みオブジェクト／動画／音声を含むスライドを自動検出し
  「温存対象」として明示（Phase 2でこれらのスライドは元データのまま保持する）
- IndexedDBに保存し、プロジェクトとして再度開ける（複数プロジェクト管理可）
- 「解析結果をJSONで書き出す」で全抽出データを確認用にダウンロード可能

## できないこと（Phase 1時点の既知の制約）

- **`.ppt`（97-2003形式）は非対応**。PowerPointで`.pptx`に変換してから使用
- テキストを含まない装飾用の図形（線・矩形など）は抽出対象外
- レイアウト/マスターからのプレースホルダ位置継承は解決しない（該当要素は
  「位置不明」としてデータ自体は保持されるが、サムネイルには表示されない）
- EMF/WMF形式の画像はブラウザで表示できないため、サムネイルには
  「プレビュー非対応形式」と表示される（バイトデータ自体は正しく保持・ハッシュ化される）
- グラフ・SmartArtの中身（系列データや図形テキスト）は解析しない。温存対象として
  検出するのみ（Phase 2で元データごと保持する設計のため、これは仕様）
- コネクタ線（矢印等）は抽出対象外

これらはすべて「データを壊す」制約ではなく「Phase 1のUIに表示しない／
Phase 2で温存扱いにする」制約です。実装引き継ぎ書§1の正直なスコーピングに対応します。

## 動作要件

- **HTTPS環境で開くこと**（GitHub Pagesならそのまま満たされる）。
  `crypto.subtle`（ハッシュ計算）はセキュアコンテキストでのみ動作するため、
  `index.html` をローカルで直接ダブルクリックして `file://` で開くと動作しません。
- 対応ブラウザ: 最新のChrome / Edge / Safari（Windows / Mac / iPad）

このZIPと同じ構成で、GitHubリポジトリに次の通りアップロードしてください
（`slide-brushup` は実際のリポジトリ名に置き換え）。

| 手元のファイル | アップロード先 |
|---|---|
| `index.html`, `package.json`, `playwright.config.js` | リポジトリ直下（`https://github.com/USER/slide-brushup/upload/main/`） |
| `libs/jszip.min.js`, `libs/pptxgen.min.js` | `.../upload/main/libs`（実装引き継ぎ書§3の手順ですでに配置済みのはず） |
| `tests/bt01〜bt04-*.spec.js` | `.../upload/main/tests` |
| `tests/fixtures/*` | `.../upload/main/tests/fixtures` |
| `.github/workflows/playwright.yml` | `.../upload/main/.github/workflows` |
| `README-phase1.md` | リポジトリ直下（任意、そのまま置いてOK） |

アップロード後、`https://USER.github.io/slide-brushup/` を開いて動作確認してください。

## 動作確認方法

1. 上記URLを開き、お手元の実際のpptx（できれば章立てや画像を含む本番に近いもの）
   をドロップ
2. スライド数・テキスト文字数・画像数・温存対象数が想定通りか確認
3. 気になるスライドの「テキストを確認」を開き、元のpptxと見比べる
4. 「解析結果をJSONで書き出す」で全データをダウンロードし、必要ならテキスト
   エディタで元資料と突き合わせる

## テストについて（GitHub Actions）

`tests/` には4種の回帰テスト（bt01〜bt04）があり、`tests/fixtures/` の
専用pptx（`make_fixtures.py` で生成したもの。python-pptxで作成し、実ブラウザの
Playwrightで実際に読み込ませて期待値と突き合わせ済み）に対して次を検証します。

- **bt01**: テキストが一字一句正しく抽出されるか／`.pptx`以外を弾くか
- **bt02**: 画像がバイト単位で一致し、同一画像が重複排除されるか
- **bt03**: スライド順がファイル名ではなくrelationships経由で正しく解決されるか
  （`basic.pptx` は意図的にファイル名の物理順と表示上の論理順をズラしてある）
- **bt04**: グラフ・SmartArtは温存対象、表・通常テキストは温存対象外と
  正しく判定されるか

ローカルNode.jsは不要です。GitHubへのpushまたはPull Requestで自動実行されます
（Actionsタブから手動実行も可能）。`libs/jszip.min.js` がまだ配置されていない
段階ではテストは失敗します（想定通りの挙動です）。
