# スライドブラッシュアップ工房 ─ Phase 1 & Phase 2

pptxインポート・解析・ハッシュ台帳・サムネイル表示（Phase 1）＋
出力エンジン・温存スライド注入・往復検証（Phase 2）。設計書§7に対応。

## Phase 2で追加されたこと

- **「pptxを書き出す」ボタン**: 解析結果（elements）からPptxGenJSで新しいpptxを再構築し、
  ダウンロードします。まだAIによるレイアウト再設計は行わず、**元と同じ内容・同じ配置**で
  1:1再構築することの正しさを先に固めるフェーズです（Phase 3以降でAIによる
  ブラッシュアップ提案が入ります）。
- **温存スライドの注入**: グラフ・SmartArt・埋め込みオブジェクト等を含むスライドは
  PptxGenJSで作り直さず、元のXMLパーツをZIPレベルでそのまま差し込みます
  （実装引き継ぎ書§5「最難関」の処理。グラフの系列データ等も完全に無傷で残ります）。
- **書き出し直後の自動検証**: 書き出したpptxを同じ解析エンジンで再度読み込み、
  ①スライド数一致 ②全文テキストハッシュ一致（無改変の証明） ③画像バイト一致
  ④温存スライドの再検出、の4点を実際にチェックして結果を表示します
  （「動くはず」ではなく実行結果で確認する方針）。
- **図形ツールの認識**（Phase 1からの修正）: テキストを持たない矩形・矢印などの
  装飾図形も、塗り色・線色・図形の種類（矩形／角丸／矢印など）込みで
  ブラッシュアップ対象として認識するようになりました。以前は「テキストを持つ図形」
  しか拾っていませんでした。

## 使い方（Phase 2）

1. Phase 1と同様にpptxを解析
2. 解析結果画面で「**pptxを書き出す（Phase 2）**」をクリック
3. 数秒〜数十秒後、検証結果が表示され、同時にファイルがダウンロードされます
4. 検証結果がすべて「OK」なら、ダウンロードされたpptxをPowerPointで開いて
   目視確認してください（グラフ・表・図形が壊れていないか等）

検証結果が1つでも「NG」の場合は、書き出し自体は完了していますが、無改変性が
崩れている可能性があります。「解析結果をJSONで書き出す」で元データと比較してください。

## Phase 2の既知の制約

- テキスト入りの図形（角丸四角形にラベル、等）は、PowerPoint上では
  「図形」と「テキストボックス」の2つの重なったオブジェクトとして再現されます
  （PptxGenJSの仕様上の制約。見た目は同じですが、後で動かす際は2つ選択が必要）
- ネストしたグループ内の回転は、グループ自身の回転との合成計算をしていません
  （単純な位置・サイズのグループ変換は正しく計算されます）
- EMF/WMF形式の画像は書き出し時にスキップされます（Phase 1同様、解析結果には
  バイトデータとして保持されていますが、pptxへの再埋め込みは非対応）
- コネクタ線（矢印線等）は直線として簡略化されます
- 再構築されたスライドは新しいデザイン（PptxGenJSの既定スタイル）になり、
  温存スライドは元の見た目のまま混在します。見た目の統一はPhase 3以降で
  AIによるデザイン提案が入った際に行われます（現時点では仕様どおりです）

## できないこと（Phase 1からの既知の制約・継続）

- **`.ppt`（97-2003形式）は非対応**。PowerPointで`.pptx`に変換してから使用
- レイアウト/マスターからのプレースホルダ位置継承は解決しない
- グラフ・SmartArtの中身（系列データや図形テキスト）は解析しない
  （温存対象として検出し、Phase 2で元データごと保持する設計のため）

## 動作要件

- **HTTPS環境で開くこと**（GitHub Pagesならそのまま満たされる）。
  `crypto.subtle`（ハッシュ計算）はセキュアコンテキストでのみ動作するため、
  `index.html` をローカルで直接ダブルクリックして `file://` で開くと動作しません。
- 対応ブラウザ: 最新のChrome / Edge / Safari（Windows / Mac / iPad）

## テストについて（GitHub Actions）

Phase 2で追加された回帰テスト:

- **bt05**: テキストなし矩形・矢印・ラベル付き図形が、塗り・線・形状タイプ込みで
  正しく認識されるか
- **bt06**: 「pptxを書き出す」の結果が、画像あり／温存スライド（グラフ）あり／
  図形ありのそれぞれのケースで、検証（テキストハッシュ・画像バイト・スライド数）
  すべて「OK」になるか

bt01〜bt04（Phase 1）と合わせて、GitHubへのpushまたはPull Requestで自動実行されます。
